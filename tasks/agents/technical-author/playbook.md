# technical-author — playbook

## Working pattern

1. Start from the audience: reviewer, non-specialist engineer, regulator, new team member.
2. Match the voice to the audience; don't dilute physics for the reviewer, don't drown a non-specialist in equations.
3. Every claim that would change a design decision must point to a result or a reference.
4. No dangling references; `docs-link-check` must pass.

## Output template

```
**Doc change:** <title>
**Audience:** <reviewer / engineer / non-specialist / regulator>
**Traceability updated:** <yes/no>
```
