# Uncertainty Quantification

## Overview

All model parameters in a radiation shielding analysis carry uncertainty — source activity, shielding thickness tolerances, material density, attenuation cross sections. Propagating these uncertainties to the dose rate estimate is essential for ALARP optimisation, safety margin justification, and honest reporting.

PyShield-SMR implements two uncertainty quantification (UQ) methods in `pyshield_smr/uq/`:

1. **Latin Hypercube Sampling (LHS)** — efficient stratified Monte Carlo for propagating parameter uncertainty.
2. **Morris Elementary Effects (EE)** — fast global sensitivity screening to identify which parameters matter most.

## Sources of Uncertainty

| Source | Typical magnitude | Type |
|---|---|---|
| Source activity (vendor certificate) | ±5–15% | Aleatoric (random) |
| Shielding thickness (manufacturing tolerance) | ±1–3 mm | Aleatoric |
| Material density (composition variability) | ±1–2% | Aleatoric |
| Buildup factor (empirical formula error) | ±5–10% | Epistemic (model) |
| Attenuation cross section (NIST XCOM accuracy) | ±0.5–2% | Epistemic |
| Monte Carlo statistical error | ~1/√N | Stochastic |

The framework treats all of these identically as input distributions — the analyst declares each uncertain parameter with a probability distribution in the YAML spec.

## Method 1: Latin Hypercube Sampling

### Theory

LHS divides the range [0, 1] for each parameter into $N$ equal strata and samples exactly one point from each stratum. This guarantees that the marginal distribution of each parameter is uniformly covered, greatly reducing the variance compared to simple Monte Carlo at the same sample count.

For $k$ uncertain parameters and $N$ samples:
1. Construct a $N \times k$ matrix where each column is a random permutation of $\{1, \ldots, N\}$.
2. Scale to uniform samples: $u_{ij} = (\pi_{ij} - r_{ij}) / N$ where $r_{ij} \sim U(0,1)$.
3. Transform to the target distributions using inverse CDFs (e.g., `scipy.stats.norm.ppf` for normal, `scipy.stats.lognorm.ppf` for lognormal).

Expected variance reduction over simple Monte Carlo: factor of ~$k$ for fully correlated parameters; 1 for fully independent. Practical speedup for $k = 5$ parameters: same accuracy with ~3–5× fewer samples.

### Implementation

`pyshield_smr/uq/monte_carlo_uq.py → LatinHypercubeSampler`:

```python
sampler = LatinHypercubeSampler(parameters, n_samples=500)
samples = sampler.generate()   # shape: (500, k)
```

Each row of `samples` is one scenario. The workflow engine runs the transport model at each scenario and collects the dose-rate vector. Statistics are then computed:

```python
doses = np.array([runner.run(scenario) for scenario in samples])
uq_result = UQResult(
    mean=doses.mean(),
    std=doses.std(),
    p05=np.percentile(doses, 5),
    p95=np.percentile(doses, 95),
    ci_95=(np.percentile(doses, 2.5), np.percentile(doses, 97.5)),
)
```

### YAML Configuration

```yaml
uncertainty:
  enabled: true
  method: lhs
  n_samples: 500
  parameters:
    - name: activity_bq
      distribution: lognormal
      median: 1.0e9
      sigma: 0.15          # 15% geometric standard deviation
    - name: thickness_m
      distribution: normal
      mean: 0.05
      std: 0.001           # ±1 mm at 1σ
    - name: density_kg_m3
      distribution: uniform
      low: 10900
      high: 11400          # Lead: 11000–11400 kg/m³ depending on purity
```

## Method 2: Morris Elementary Effects

### Theory

The Morris method identifies **which parameters have the largest influence on the output** using a one-at-a-time (OAT) perturbation approach. For each parameter $x_i$, an **elementary effect** is:

$$EE_i = \frac{y(x_1, \ldots, x_i + \Delta, \ldots, x_k) - y(\mathbf{x})}{\Delta}$$

where $\Delta$ is a fixed perturbation step. By sampling $r$ random base points and computing $EE_i$ at each, the **mean** $\mu_i^*$ (= mean of |$EE_i$|) and **standard deviation** $\sigma_i$ screen parameters:

- High $\mu_i^*$ → parameter has a large influence on output (important).
- High $\sigma_i$ / $\mu_i^*$ → influence is nonlinear or interacts with other parameters.

Total model evaluations: $r(k+1)$, typically 10–30× cheaper than full LHS for the same screening resolution.

### Implementation

`pyshield_smr/uq/sensitivity.py → MorrisSampler`:

```python
morris = MorrisSampler(parameters, n_trajectories=20, n_levels=4)
trajectories = morris.generate()         # shape: (20*(k+1), k)
doses = np.array([runner.run(t) for t in trajectories])
mu_star, sigma = morris.analyse(doses)
```

Output ranks parameters by $\mu_i^*$; the top 2–3 parameters typically account for >80% of output variance.

### When to Use Each Method

| Goal | Method | Reason |
|---|---|---|
| Full uncertainty distribution of dose rate | LHS (N ≥ 500) | Samples full distribution |
| Identify the 2–3 most important parameters | Morris EE | 10× cheaper than LHS |
| Regulatory submission (conservative bounds) | LHS + 95th percentile | Provides probability statement |
| ALARP design loop (many evaluations) | Morris EE first, then LHS on top params | Cost-efficient |

## Reporting

UQ results are written to `reports/<case>/uq_summary.json`:

```json
{
  "method": "lhs",
  "n_samples": 500,
  "dose_rate_sv_per_h": {
    "mean": 1.23e-5,
    "std": 2.1e-6,
    "p05": 8.5e-6,
    "p95": 1.7e-5,
    "ci_95": [7.9e-6, 1.8e-5]
  },
  "sensitivity": {
    "activity_bq": {"mu_star": 0.82},
    "thickness_m": {"mu_star": 0.14},
    "density_kg_m3": {"mu_star": 0.04}
  }
}
```

The HTML report includes: distribution histogram of dose rate, sensitivity bar chart, and a statement of the 95th percentile dose rate for zone assignment.
